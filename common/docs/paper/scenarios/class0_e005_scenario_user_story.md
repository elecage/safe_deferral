# Class 0 E005 Scenario User Story
## 실내 낙상 감지로 긴급 대응이 필요한 상황

## 1. 이 문서가 설명하는 JSON 파일

이 문서는 다음 scenario JSON 파일을 실제 사용환경에서 일어날 수 있는 상황으로 풀어 설명한다.

```text
integration/scenarios/class0_e005_scenario_skeleton.json
```

이 JSON 파일은 집 안에서 **사용자의 낙상 또는 낙상 의심 상황이 감지되었을 때**, 시스템이 이를 긴급상황으로 인식하고 즉시 대응 흐름으로 전환하는지를 검토하기 위한 시나리오다.

여기서는 사용자가 침대, 휠체어, 욕실, 거실 등에서 균형을 잃고 넘어졌거나, 넘어졌을 가능성이 있는 상황에서 Fall Detection Node 또는 Emergency Node가 낙상 감지 정보를 전달하는 상황을 예로 든다.

이 문서는 이후 실제 노드를 개발할 때 다음을 확인하는 기준으로도 사용할 수 있다.

```text
- 어떤 노드가 낙상 또는 낙상 의심 상황을 감지해야 하는가
- 낙상 감지 정보는 어떤 방식으로 중앙 시스템에 전달되는가
- 시스템은 일반 생활 요청과 낙상 긴급상황을 어떻게 구분하는가
- 사용자가 직접 도움을 요청하지 못하는 상황에서 어떤 결과가 기대되는가
```

---

## 2. 발생 상황

사용자는 집 안에서 이동하거나 자세를 바꾸는 중이라고 가정한다.

예를 들어 사용자는 침대에서 휠체어로 옮겨 앉으려 하거나, 욕실에서 방향을 바꾸거나, 거실에서 보행 보조기기를 사용해 이동하고 있을 수 있다.

이때 사용자가 균형을 잃고 넘어지거나, 바닥에 주저앉거나, 일정 시간 이상 비정상적인 자세로 머무르는 상황이 발생한다.

사용자는 넘어진 직후 통증, 당황, 어지러움, 근력 저하 등으로 인해 스마트폰을 조작하거나 음성으로 도움을 요청하기 어려울 수 있다. 특히 혼자 있는 상황이라면 외부에서 사용자의 상태를 바로 알기 어렵다.

따라서 시스템은 낙상 감지 정보를 일반적인 생활 보조 요청이 아니라 **긴급상황**으로 판단해야 한다.

이 시나리오는 E001 고온 이상, E003 연기 감지, E004 가스 감지와 같은 Class 0 긴급상황이다. 다만 앞선 시나리오들이 주로 환경 위험을 감지하는 상황이라면, E005는 **사용자의 신체 상태 변화 또는 이동 중 사고**를 감지하는 상황이라는 점에서 차이가 있다.

이 시나리오의 핵심 상황은 다음과 같다.

```text
사용자가 실내에서 넘어졌거나 낙상 의심 상태가 감지되고,
사용자는 직접 도움을 요청하기 어려울 수 있으며,
시스템은 이를 긴급상황으로 인식해야 한다.
```

---

## 3. 시스템이 인식해야 하는 상황

시스템은 사용자의 버튼 입력이나 음성 요청이 없어도, 낙상 감지 노드가 제공하는 정보를 확인해야 한다.

예를 들어 시스템은 다음과 같은 정보를 함께 확인한다.

```text
- 사용자의 낙상 또는 낙상 의심 상태가 감지되었다.
- 낙상 감지 정보가 긴급상황 감지 노드에서 전달되었다.
- 사용자가 직접 스마트폰을 조작하거나 음성으로 도움을 요청하지 못할 수 있다.
- 일반적인 조명 제어 요청이나 생활 보조 요청이 아니다.
- 즉각적인 사용자 확인, 경고, 보호자 알림이 필요할 수 있다.
```

이 정보를 종합하면, 시스템은 이 상황을 **낙상 감지에 따른 긴급상황**으로 처리해야 한다.

여기서 중요한 점은 낙상 감지 정보가 시스템 안에 저절로 존재하는 것이 아니라는 점이다. 낙상 정보는 Fall Detection Node, Wearable Sensor Node, Motion Sensor Node, Vision/Depth Sensor Node, 또는 Emergency Node가 실제로 감지하고 전달해야 한다.

Mac mini Edge Hub는 이 긴급상황 정보를 수신한 뒤, 일반적인 사용자 요청 처리 흐름보다 긴급 대응 흐름을 우선해야 한다.

## 4. 정보 출처 노드 및 상태 정보

이 시나리오에서 시스템은 낙상 감지 노드가 제공하는 정보를 중심으로 현재 상황을 판단한다. 동시에 사용자 상태, 공간 정보, 안내 출력, 보호자 알림, 기록 기능도 함께 고려된다.

| 항목 | 정보를 제공하는 노드/구성요소 | 제공되는 정보 | 시나리오 내 상태 |
|---|---|---|---|
| 낙상 감지 | Fall Detection Node 또는 Emergency Node | 사용자의 낙상 또는 낙상 의심 감지 정보 | 낙상 감지됨 |
| 사용자 움직임 | Wearable Sensor Node, Motion Sensor Node 또는 Vision/Depth Sensor Node | 급격한 자세 변화, 충격, 비정상 정지 상태, 바닥 근접 상태 등 | 비정상 움직임 또는 자세 감지 |
| 긴급상황 이벤트 | Emergency Node | 낙상 감지가 긴급상황에 해당한다는 이벤트 | 긴급 이벤트 발생 |
| 사용자 상태 | Bounded Input Node 또는 사용자 상태 정보 | 사용자가 직접 도움을 요청하거나 취소할 수 있는지와 관련된 입력/상태 | 직접 조작이 없거나 제한적 조작 가능 상태 |
| 환경 정보 | Context Node | 사용자가 있는 공간, 주변 환경, 점유 여부 등 | 실내 공간에서 낙상 의심 상황 발생 |
| 방문자 여부 | Doorbell/Visitor Context Node | 현관 호출 또는 방문자 감지 정보 | 방문자 상황 아님 |
| 요청/상황 해석 | Mac mini Edge Hub | 낙상 정보를 종합한 처리 판단 | 낙상 긴급상황으로 해석 |
| 사용자 확인/경고 | TTS/Voice Output 또는 Warning Output Node | 사용자에게 상태 확인 또는 경고 안내 | 확인/경고 안내 가능 |
| 보호자/외부 알림 | Caregiver Notification 또는 Escalation Interface | 보호자나 관리자에게 낙상 상황 전달 | 필요 시 긴급 알림 가능 |
| 처리 기록 | Audit Log | 감지 정보, 처리 판단, 알림 결과 기록 | 긴급상황 처리 결과 기록 |

이 표에서 낙상 감지 상태는 단순한 문서상의 상태가 아니라, 실제 센서 노드가 제공하는 위험 신호다.

```text
낙상 감지 정보
→ Fall Detection Node, Wearable Sensor Node, Motion Sensor Node, Vision/Depth Sensor Node 또는 Emergency Node가 제공하는 긴급상황 정보

긴급상황 처리 판단
→ Mac mini Edge Hub가 낙상 감지 정보를 바탕으로 수행하는 처리 결과
```

따라서 실제 개발에서는 “낙상이 감지되었다”는 상태를 단순히 문서에 적는 것이 아니라, **어떤 센서 노드가 사용자의 움직임, 자세 변화, 충격, 비정상 정지 상태를 감지하고, 그 정보를 어떤 방식으로 긴급 이벤트로 전달할 것인가**까지 설계해야 한다.

## 5. 노드별 개발 관점 정리

이 시나리오를 구현하기 위해 필요한 노드와 구성요소는 다음과 같다.

| 노드/구성요소 | 개발 시 필요한 기능 | 이 시나리오에서의 역할 |
|---|---|---|
| Fall Detection Node | 낙상 또는 낙상 의심 상태 감지, 이벤트 생성 | 사용자의 낙상 상황을 감지 |
| Wearable Sensor Node | 가속도, 자세 변화, 충격, 움직임 정지 상태 감지 | 사용자의 급격한 움직임 변화나 충격을 감지 |
| Motion Sensor Node | 실내 움직임 변화, 장시간 움직임 없음 감지 | 낙상 후 비정상 정지 상태를 보조적으로 확인 |
| Vision/Depth Sensor Node | 바닥 근접 자세, 쓰러진 자세, 움직임 패턴 감지 | 카메라 또는 깊이 센서 기반 낙상 확인에 사용 가능 |
| Emergency Node | 낙상, 가스, 연기, 고온 등 긴급상황 이벤트 생성 | 낙상 감지를 긴급 이벤트로 전달 |
| Context Node | 공간 정보, 주변 환경 정보 제공 | 낙상이 발생한 공간과 주변 상태를 이해하는 데 도움 |
| Bounded Input Node | 사용자의 수동 입력 또는 취소 입력 감지 | 사용자가 도움 요청을 취소하거나 추가 입력을 줄 수 있는 경우 사용 |
| Doorbell/Visitor Context Node | 현관 호출 또는 방문자 감지 여부 제공 | 이 시나리오에서는 방문자 상황이 아님을 나타냄 |
| Mac mini Edge Hub | 긴급 이벤트 수신, 상황 판단, 긴급 대응 흐름 실행 | 낙상 감지를 Class 0 긴급상황으로 처리 |
| TTS/Voice Output | 사용자에게 상태 확인 또는 위험 상황을 음성으로 안내 | 필요 시 “괜찮으신가요?” 또는 “도움 요청을 보냅니다.” 안내 |
| Warning Output Node | 경고음, 표시등, 화면 알림 등 보조 경고 출력 | 주변 사람에게 상황을 알리거나 사용자 확인을 도움 |
| Caregiver Notification 또는 Escalation Interface | 보호자나 관리자에게 알림 전송 | 사용자의 낙상 또는 낙상 의심 상황을 외부에 전달 |
| Audit Log | 감지, 판단, 알림, 처리 결과 기록 | 긴급상황 처리 과정 기록 |

## 6. 처리 과정

### 6.1 낙상 또는 낙상 의심 상태가 감지된다

Fall Detection Node, Wearable Sensor Node, Motion Sensor Node 또는 Vision/Depth Sensor Node가 사용자의 낙상 또는 낙상 의심 상태를 감지한다.

이 시나리오에서는 사용자가 이동 중 균형을 잃었거나, 바닥에 쓰러졌거나, 갑작스러운 충격 이후 일정 시간 움직임이 없는 상황을 가정한다.

예를 들어 다음과 같은 의미의 이벤트가 생성된다.

```text
실내에서 사용자 낙상 또는 낙상 의심 상태 감지
```

---

### 6.2 긴급상황 이벤트가 생성된다

Emergency Node는 감지된 낙상 정보를 긴급상황 이벤트로 구성한다.

이 이벤트는 일반적인 사용자 요청이 아니라, 즉각적인 확인과 대응이 필요한 위험 신호로 취급된다.

이 시나리오에서 긴급상황 이벤트는 다음 입력 경로로 전달된다.

```text
safe_deferral/emergency/event
```

---

### 6.3 Mac mini Edge Hub가 긴급 이벤트를 수신한다

Mac mini Edge Hub는 Emergency Node가 보낸 낙상 감지 긴급 이벤트를 수신한다.

이때 시스템은 사용자가 버튼을 눌렀는지, 음성 명령을 했는지와 무관하게 낙상 감지 이벤트를 우선 처리해야 한다.

즉, 이 시나리오는 사용자의 일상 요청을 해석하는 흐름이 아니라, 사용자의 신체 상태 변화나 사고 가능성을 우선 처리하는 흐름이다.

---

### 6.4 시스템이 낙상 긴급상황으로 판단한다

Mac mini Edge Hub는 낙상 감지 정보를 바탕으로 현재 상황을 긴급상황으로 판단한다.

이때 시스템은 다음과 같은 점을 함께 고려할 수 있다.

```text
낙상 또는 낙상 의심 상태가 감지되었다.
사용자가 직접 도움을 요청하지 못할 수 있다.
움직임이 제한되었거나 통증으로 반응이 어려울 수 있다.
즉각적인 사용자 확인 또는 보호자 알림이 필요할 수 있다.
```

이 시나리오에서는 이 상황을 낙상 감지에 따른 긴급상황으로 해석한다.

---

### 6.5 사용자에게 상태 확인 또는 경고를 제공한다

필요한 경우 TTS/Voice Output 또는 Warning Output Node가 사용자에게 상태 확인 메시지를 제공한다.

예를 들어 다음과 같은 안내가 가능하다.

```text
“낙상이 감지되었습니다. 괜찮으신가요?”
```

또는 다음과 같은 후속 안내가 가능하다.

```text
“응답이 없으면 보호자에게 알림을 보냅니다.”
```

사용자가 시각·청각·운동 능력에 제한이 있을 수 있으므로, 안내 방식은 음성, 경고음, 표시등, 화면 알림 등 여러 형태로 제공될 수 있어야 한다.

---

### 6.6 사용자의 응답 여부를 확인할 수 있다

시스템은 짧은 시간 동안 사용자의 응답을 확인할 수 있다.

예를 들어 사용자가 버튼을 누르거나 음성으로 “괜찮다”고 답하거나, 움직임이 정상적으로 회복되는 경우를 확인할 수 있다.

그러나 사용자가 응답하지 못하거나, 움직임이 계속 감지되지 않거나, 상황이 불확실하다면 시스템은 보호자 또는 관리자 알림으로 이어질 수 있다.

이 단계는 실제 구현 방식에 따라 달라질 수 있지만, 사용자에게 불필요한 공포를 주지 않으면서도 긴급 도움 연결을 지연하지 않는 방식으로 설계되어야 한다.

---

### 6.7 보호자 또는 관리자에게 알림을 보낼 수 있다

사용자가 직접 상황을 설명하거나 도움을 요청하기 어렵다고 판단되는 경우, 시스템은 보호자나 관리자에게 알림을 보낼 수 있다.

예를 들어 다음과 같은 정보가 전달될 수 있다.

```text
- 낙상 또는 낙상 의심이 감지된 시간
- 낙상이 감지된 위치 또는 연결된 노드
- 감지된 움직임 변화 또는 충격 상태
- 사용자 응답 여부
- 시스템이 긴급상황으로 처리했다는 사실
```

이 단계는 사용자가 직접 전화를 걸거나 메시지를 보내기 어려운 상황에서 외부 도움을 빠르게 연결하기 위한 것이다.

---

### 6.8 처리 결과를 기록한다

Audit Log는 이 과정에서 발생한 주요 정보를 기록한다.

예를 들어 다음과 같은 내용이 기록될 수 있다.

```text
- 낙상 이벤트가 발생한 시간
- 이벤트를 제공한 노드
- 감지된 움직임 변화 또는 자세 상태
- 시스템이 낙상 긴급상황으로 판단한 내용
- 사용자에게 확인 안내가 제공되었는지 여부
- 사용자 응답 여부
- 보호자 또는 관리자에게 알림이 전송되었는지 여부
- 이후 처리 결과
```

이 기록은 나중에 시스템이 정상적으로 긴급상황을 처리했는지 확인하거나, 실험 결과를 분석할 때 사용할 수 있다.

---

## 7. 예상되는 결과

이 시나리오에서 기대되는 결과는 다음과 같다.

낙상 또는 낙상 의심 상태가 감지되면 시스템은 이를 일반적인 생활 보조 요청으로 처리하지 않고, 긴급상황으로 인식한다.

사용자 입장에서는 다음과 같은 흐름이 된다.

```text
사용자가 실내에서 넘어졌거나 낙상 의심 상태가 감지된다.
→ Fall Detection Node 또는 Emergency Node가 긴급 이벤트를 생성한다.
→ Mac mini Edge Hub가 긴급 이벤트를 수신한다.
→ 시스템이 낙상 감지 상황을 긴급상황으로 판단한다.
→ 사용자에게 상태 확인 또는 경고가 제공된다.
→ 응답이 없거나 위험이 계속되면 보호자나 관리자에게 알림이 전달된다.
→ 처리 과정이 기록된다.
```

## 8. 사용자에게 주는 의미

이 시나리오는 사용자가 직접 도움을 요청하기 어려운 낙상 상황에서 중요하다.

지체장애인, 고령자, 또는 이동 능력이 제한된 사용자는 넘어졌을 때 스스로 일어나기 어렵거나, 통증과 당황으로 인해 즉시 도움을 요청하지 못할 수 있다.

특히 욕실, 침실, 거실처럼 일상적으로 이동하는 공간에서도 낙상은 갑자기 발생할 수 있다. 혼자 있는 상황이라면 낙상 사실이 늦게 발견될 수 있고, 이로 인해 위험이 커질 수 있다.

이때 시스템이 먼저 낙상 또는 낙상 의심 상태를 감지하고, 사용자에게 확인을 시도하거나 보호자에게 알림을 보내면 대응 시간을 줄일 수 있다.

이러한 방식은 특히 다음과 같은 사용자에게 도움이 될 수 있다.

```text
- 침대에서 휠체어로 이동할 때 균형을 잃기 쉬운 사용자
- 욕실이나 거실에서 낙상 위험이 높은 고령자
- 넘어졌을 때 스스로 일어나기 어려운 사용자
- 통증, 어지러움, 당황으로 도움을 요청하기 어려운 사용자
- 혼자 있을 때 낙상 사실이 늦게 발견될 위험이 있는 사용자
```

## 9. 개발 관점에서 확인해야 할 점

이 문서는 실제 상황 설명을 중심으로 하지만, 노드 개발을 위해서는 다음 사항을 확인해야 한다.

```text
- Fall Detection Node는 낙상 또는 낙상 의심 상태를 안정적으로 감지할 수 있는가?
- Wearable Sensor Node는 충격, 자세 변화, 움직임 정지 상태를 제공할 수 있는가?
- Motion Sensor Node 또는 Vision/Depth Sensor Node는 실내 움직임 변화와 비정상 자세를 보조적으로 확인할 수 있는가?
- Emergency Node는 낙상 감지 상황을 긴급 이벤트로 변환할 수 있는가?
- Mac mini Edge Hub는 낙상 감지 이벤트를 일반 입력보다 우선 처리할 수 있는가?
- TTS/Voice Output은 사용자에게 상태 확인 메시지를 즉시 안내할 수 있는가?
- Warning Output Node는 음성 외의 경고 방식을 제공할 수 있는가?
- Bounded Input Node는 사용자의 취소 또는 확인 입력을 받을 수 있는가?
- Caregiver Notification 또는 Escalation Interface는 보호자에게 필요한 정보를 전달할 수 있는가?
- Audit Log는 감지, 판단, 사용자 응답, 알림, 처리 결과를 빠짐없이 기록할 수 있는가?
```

이 체크리스트는 낙상 감지 긴급상황을 실제 시스템으로 구현할 때 필요한 최소 개발 항목을 정리한 것이다.

## 10. 이 시나리오의 핵심

이 시나리오의 핵심은 다음과 같다.

```text
사용자가 직접 요청하지 못하더라도,
긴급상황 감지 노드가 낙상 또는 낙상 의심 상태를 감지하면,
Mac mini Edge Hub가 이를 우선 처리하고,
사용자 확인과 보호자 알림으로 연결한다.
```

즉, 이 시나리오는 스마트홈 시스템이 사용자의 제한된 조작 능력에만 의존하지 않고, 낙상과 같은 신체적 위험 상황을 먼저 파악해 안전 대응으로 연결하는 기본 긴급상황 시나리오다.
